# SOC Metrics & Objectives

This project documents my learning from the SOC Metrics & Objectives lab, focusing on how Security Operations Centers measure performance, efficiency, and detection capability.

---

## 🎯 Purpose of SOC Metrics

SOC metrics are essential to measure how effectively a Security Operations Center detects, investigates, and responds to threats.

Without metrics, a SOC cannot:
- Measure detection quality
- Improve response time
- Reduce false positives
- Demonstrate business value

---

## 📊 Key SOC Metrics

### 1️⃣ Mean Time To Detect (MTTD)
The average time taken to identify a potential security incident.

Formula:
Total Detection Time / Number of Incidents

---

### 2️⃣ Mean Time To Respond (MTTR)
The average time required to respond and contain an incident.

Formula:
Total Response Time / Number of Incidents

---

### 3️⃣ False Positive Rate
Percentage of alerts incorrectly identified as threats.

High false positives can:
- Overload analysts
- Reduce efficiency
- Increase burnout risk

---

### 4️⃣ Alert Volume
Total number of alerts generated within a given timeframe.

This helps measure:
- Detection rule noise
- Infrastructure risk exposure
- SOC workload

---

## 🧠 Why This Matters

Understanding SOC metrics helps analysts:

- Improve detection logic
- Reduce alert fatigue
- Optimize investigation workflows
- Align security operations with business goals
